## 双Agent赛马实验框架

### 一、整体架构设计

**实验分组结构**

```
账户下计划池
├── UAA组（Bucket A）          → 快手UAA托管，内部Agent熔断
├── 内部Agent组（Bucket B）    → 内部Agent托管，UAA不开启
└── 常规对照组（Bucket C）     → 无任何智能调控，人工基线
```

三组之间**计划级互斥**，一个计划只归属一个Bucket，全程不漂移。

---

### 二、策略互斥机制设计

**熔断/旁路逻辑**

```
内部Agent执行前置检查：
  IF 计划.uaa_status == ACTIVE
    THEN 跳过本计划，写入日志（action=bypass, reason=uaa_active）
    ELSE 执行内部Agent调控逻辑
```

|场景|UAA状态|内部Agent行为|备注|
|---|---|---|---|
|正常实验|OFF|正常执行|Bucket B标准态|
|UAA组|ON|强制旁路|Bucket A标准态|
|异常切换（计划误开UAA）|ON→切换|立即熔断，等待下一周期重判|保护机制|
|UAA异常下线|OFF|不自动接管，人工确认后才切换|防止调控真空期抖动|

**熔断触发后的日志字段（用于归因排查）**

```json
{
  "plan_id": "xxx",
  "bucket": "B",
  "action": "bypass",
  "reason": "uaa_active_detected",
  "ts": "2024-xx-xx xx:xx:xx",
  "agent_intended_action": "price_down_5pct"
}
```

### 三、流量分配方案

**分配原则**

随机分配 + 分层抽样，保证三组在以下维度分布一致：

- 行业/品类
- 历史消耗区间（高/中/低预算段）
- 历史转化成本区间
- 计划创建时间（排除新老计划偏差）

**推荐分组比例**

|实验组|比例|说明|
|---|---|---|
|UAA组（A）|40%|主要被测方|
|内部Agent组（B）|40%|主要被测方|
|常规对照组（C）|20%|基线锚点，不宜过少|

对照组 20% 是最低保障，若账户计划总量 < 30个，建议扩大至 1:1:1 均分，保证每组样本量。

**分配执行方式**

以计划ID做哈希取模，保证分配稳定不漂移：

```python
bucket = hash(plan_id + experiment_id) % 100
if bucket < 40:   assign("A")  # UAA组
elif bucket < 80: assign("B")  # 内部Agent组
else:             assign("C")  # 对照组
```

### 四、核心度量指标

**主指标（Primary Metrics）**

|指标|计算方式|说明|
|---|---|---|
|相同消耗下转化数|Conversions / Spend（归一化）|核心赛马指标，消除预算差异|
|转化成本（CAC）|Spend / Conversions|越低越好|
|ROI|Revenue / Spend|有交易数据时使用|

**辅助指标（Guardrail Metrics）**

|指标|用途|
|---|---|
|成本达标率|各组成本稳定在目标±15%以内的计划比例|
|消耗达成率|实际消耗 / 预算目标，防止某组因调控过激导致欠跑|
|调控动作频次|对比UAA和内部Agent的介入频率是否合理|
|计划存活率|实验周期内未被关停的计划比例|

**不纳入评估的情况（排除规则）**

- 冷启期内的计划（定义：开始投放后转化数 < N 个，或运行 < 3天）
- 实验期间被人工干预过出价/预算的计划
- 账户级别发生大规模预算调整的周期段

### 五、样本量与检验设计

**最小样本计算逻辑**

基于双侧 t 检验：

```
所需计划数 n = 2 × (Z_α/2 + Z_β)² × σ² / δ²

参数建议取值：
  显著性水平 α = 0.05  →  Z_α/2 = 1.96
  统计功效   β = 0.80  →  Z_β   = 0.84
  最小可检测提升 δ = 10%（相对于对照组均值）
  σ 取历史转化成本的标准差
```

**实际工程约束**

|条件|最低门槛|说明|
|---|---|---|
|每组计划数|≥ 30 个|保证统计稳定性|
|每组日消耗|≥ 5万元|保证转化量可观测|
|每组累计转化数|≥ 200 个|避免小样本波动|
|实验持续周期|≥ 14天（冷启后起算）|覆盖至少2个完整投放节奏周期|

若以上任一条件未达到，该组不进入当期评估，延长观察。

### 六、实验生命周期

```
T0  计划哈希分配 → 三组锁定，写入实验配置表
 │
T1  冷启期（约3~7天）
 │  ├── UAA组：UAA自动基建 + 自动调控
 │  ├── Agent组：内部Agent运行，UAA旁路检查激活
 │  └── 对照组：人工设置后不干预
 │
T2  冷启期结束判断（转化数达标 + 消耗稳定）
 │  └── 未达标计划继续等待，不提前纳入统计
 │
T3  正式评估期（≥14天）
 │  ├── 每日记录：消耗、转化数、成本、调控动作日志
 │  ├── 每周中检：异常计划排查，不调整分组
 │  └── 终止规则：若某组成本超出目标50%且连续3天，提前终止该组保护预算
 │
T4  结果分析
     ├── 主指标显著性检验（t检验 / Bootstrap）
     ├── 辅助指标归因分析
     └── 分层分析（按行业、预算段看差异来源）
```

### 七、结果判断标准

|结论|判定条件|
|---|---|
|UAA显著优于内部Agent|A组主指标显著优于B组（p<0.05），且优于对照组C|
|内部Agent显著优于UAA|B组主指标显著优于A组（p<0.05），且优于对照组C|
|两者无显著差异|A/B组主指标差异 p>0.05，看辅助指标做业务判断|
|双Agent均不如对照|A、B均差于C，说明调控策略有问题，暂停迭代排查|

**分层分析优先级**：主指标无显著差异时，按行业/预算段拆分看是否存在结构性差异，指导后续差异化策略（高预算计划用UAA，长尾计划用内部Agent）。

---

### 八、关键风险与应对

|风险|应对方案|
|---|---|
|对照组被污染（人工误调控）|对照组计划打"只读"标记，系统层面拦截人工批量操作|
|UAA悄默更新调控逻辑导致实验中断|监控UAA版本变更日志，版本切换视为实验重启信号|
|计划自然淘汰（跑量差被主动关停）|记录计划退出时间，分析是否与调控行为相关，纳入存活率指标|
|分组后行业分布不均|T0分配后立即做分布校验，偏差>10%则重新哈希|
