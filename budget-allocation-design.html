信息流广告预算智能分配系统

设计方案 · v1.3  |  覆盖渠道：头条 / 广点通 / 快手  |  投放目标：APP 下载新客获取  |  新增：固定成本约束下的预算-转化曲线

系统概述

本方案针对信息流渠道 APP 下载拉新链路中两个核心痛点，设计一套基于算法的预算智能分配系统：

  • 渠道内账户不均衡：同一渠道下不同账户跑量效果差异显著，需基于渠道视角合理分配各账户预算；
  • 渠道间 ROI 不均衡：全局大盘不同渠道 ROI 效果不同，需基于全局视角动态分配渠道预算。

核心建设思路：将预算分配抽象为流量池最优化分解问题，通过建立"预算 - 成本 - 转化"三者的预估模型，将充沛预算优先分配给能跑量、能带来有效目标转化的账户和计划。建模时必须锚定一个前提：成本/CPA 是固定外生约束，不随预算变化而变化；系统衡量的是在固定成本约束下，预算变化带来的转化数变化，而不是预算和成本同时变化后的混合结果。

目标函数：在总预算与固定 CPA 成本约束下，最大化全链路目标转化数（新客 APP 下载激活）。其中 CPA 不作为优化变量，预算优化只解释转化数随预算变化的响应曲线。
识别边界：若 CPA、出价或成交成本同时发生变化，则转化波动无法单独归因于预算调节。此类样本需进入协变量控制、分层分析或剔除，不参与主响应曲线拟合。

整体架构

采用两级优化结构：渠道级（全局视角）先行，账户级（渠道内视角）次之，两级共享相同的 S 曲线优化框架。

数据层
账户消耗历史（1 日 / 3 日 / 7 日窗口）· 转化数据（CPA 成本、转化数）· 渠道 ROI 汇总 · 平台竞争度指数
模型层
S 曲线参数拟合（每账户 T+1 凌晨更新，参数 L / r / B₀）· 花费预估模型（消耗率 × 竞争度指数）
优化层
渠道级预算分配（全局等边际效率 · 二分迭代求解 λ）· 账户级预算分配（渠道内二分迭代，约束 [B_min, B_max]）
执行层
日预算写入调度（每日 00:30 执行）· 盘中预算动态调整（可选，每 4 小时一次）· 投放平台 API 推送

模块一：预算约束生成

每个账户的日预算取值范围 [B_min, B_max] 由三个历史消耗窗口共同决定，兼顾短期波动与长期运行稳定性：

窗口数据来源在约束中的作用
1 日（D-1）昨日实际消耗短期基准，防止单日突变
3 日近 3 日消耗均值 / 最大值中期稳定性参考，平滑偶发噪声
7 日近 7 日消耗均值 / 最大值长期运行上界，防止过度扩张

约束计算公式

预算下界（最小保障）
B_min = max( P10(近7日消耗), 3日均值 × 0.5 )
预算上界（最大允许）
B_max = min( 3日最大值 × 1.5, 7日最大值 × 1.2 )
冷启动兜底：若账户近 7 日有效消耗数据不足（新账户或冷启动期），B_min 取平台最低保护预算（建议 ≥500 元/日），B_max 取同渠道同类账户消耗分布的 P50 水位，避免新账户在竞争中被完全剥夺预算。

模块二："预算 - 成本 - 转化" S 曲线预估模型

核心假设：在固定 CPA 目标成本约束下，成本是外生常量，目标转化数随预算增大呈 S 型增长——初期边际效率高（预算充裕、流量充足），后期因可触达优质流量逐步消耗，边际效率递减，最终趋于饱和（流量容量上限 L）。因此，本模块拟合的是 Conv(B | CPA = CPA_fixed)，而不是同时让预算 B 与成本 CPA 共同变化。

成本固定约束：只衡量预算对转化的影响

建模对象
Conv = f(B | CPA = CPA_fixed, bid = bid_fixed, target = target_fixed)
CPA、出价目标、投放目标、人群定向等作为固定条件或控制变量进入模型；主曲线只描述预算 B 增减时，转化数 Conv 的响应关系。
不纳入主曲线的混合效应
ΔConv ≠ f(ΔB, ΔCPA) 的混合解释
如果 CPA 同时下降或上升，转化变化可能来自成本改善/恶化，而非预算分配本身。此时不能直接用该样本拟合预算响应曲线。

S 曲线转化预估公式

转化数预估（固定成本约束下的 Logistic S 曲线）
Conv(B | CPA_fixed) = L / ( 1 + exp( −r · (B − B₀) ) )
L ：固定 CPA 约束下的转化容量上限（历史最大转化 × 放大系数 α，建议 α = 1.2）
r ：增长斜率（拟合参数，反映预算对转化的敏感程度，不解释 CPA 变化）
B₀ ：S 曲线拐点预算（固定成本下边际效率最大处，即效率斜率 k 取最大值对应的预算点）

边际效率函数（一阶导数）

边际转化效率 ME(B)
ME(B) = dConv/dB = L·r·exp(−r·(B−B₀)) / (1 + exp(−r·(B−B₀)))²
ME(B) 在 B = B₀ 处取得最大值 L·r/4，此后随预算增大单调递减（边际递减效应）。

平均效率指标 k（ROI 斜率）

效率斜率 k(B) = 目标转化数 / 账户预算
k(B) = Conv(B) / B
最优预算点 B* 满足 k'(B*) = 0，即：ME(B*) · B* = Conv(B*)
B* 是 S 曲线上性价比最高的预算量——边际增益等于当前平均增益的交叉点。
预算低于 B*：增加预算可提升 k（物超所值区）；超过 B*：增加预算 k 下降（边际递减区）。
参数拟合方法：以账户近 30 日「日预算 → 日转化数」历史数据对，通过最小二乘法（Levenberg-Marquardt 算法）拟合 L、r、B₀ 三个参数，每日 T+1 凌晨更新一次。若有效数据点不足 7 个，退化为线性外推模型（Conv ≈ k̄·B，k̄ 为近期平均效率）。

模块三：两级最优化分配

3.1 理论基础：等边际效率原则（拉格朗日乘子法）

对于集合内任意分配单元（账户或渠道），在总预算等式约束与各单元预算区间约束下，求解最优分配：

优化目标
max Σᵢ Conv_i(Bᵢ)
s.t. Σᵢ Bᵢ = B_total
B_min,i ≤ Bᵢ ≤ B_max,i ∀ i
KKT 最优条件（等边际效率）
ME_i(Bᵢ*) = ME_j(Bⱼ*) = λ ∀ i,j（未触及约束边界的单元间）
直觉：若账户 A 的边际转化效率 > 账户 B，则从 B 抽取预算转移给 A 可提升总转化；持续调整直至两者边际效率相等，此时总转化数达到最大。

3.2 求解算法：二分法迭代（无需 LP 库）

# 输入：各分配单元的 S 曲线参数、约束区间 [B_min,B_max]、总预算 B_total
def allocate(units, B_total):

    # 给定 λ，对单元 u 反解预算（ME(B)=λ 的解，截断至约束区间）
    def solve_B(u, lam):
        B = ME_inverse(u.L, u.r, u.B0, lam)  # 解析式或数值求逆
        return clip(B, u.B_min, u.B_max)

    # 二分搜索 λ 使 ΣB_i = B_total
    lo, hi = 0, MAX_ME
    while hi - lo > EPS:
        lam = (lo + hi) / 2
        total = sum(solve_B(u, lam) for u in units)
        if total > B_total:
            lo = lam   # λ 太小 → 各单元分得预算过多 → 提高 λ
        else:
            hi = lam   # λ 太大 → 各单元分得预算不足 → 降低 λ

    return {u.id: solve_B(u, (lo+hi)/2) for u in units}

收敛条件：|ΣBᵢ − B_total| < 100 元，或迭代超 50 次取最近解。时间复杂度 O(N·log(MAX_ME/EPS))。

3.3 两级分配执行顺序

级别优化对象约束输入输出结果
第一级 渠道级 头条 / 广点通 / 快手 三渠道 各渠道历史预算上下界；总日预算 B_total 各渠道当日预算额度 B_channel
第二级 账户级 渠道内所有账户（每渠道独立求解） 各账户 [B_min, B_max]；渠道预算 B_channel 各账户当日预算 B_account
渠道聚合曲线构建：渠道级 S 曲线 = 渠道内所有账户 S 曲线的垂直叠加（在相同 λ 水平下，各账户最优预算之和 vs. 渠道总转化数），保证两级分配结果全局自洽。

模块四：花费预估与校验调整

在确定理论最优预算后，引入花费预估服务验证预算的实际可消耗性，防止"名义高预算但实际跑不出去"导致的预算浪费。

预估模型输入与输出

输入因子说明获取方式
历史消耗率 η_a近 7 日「实际消耗 / 分配预算」均值，反映账户跑量能力内部数据仓库
竞争度系数 γ渠道当日流量竞争水位（影响出价有效性）平台 API 或内部模型估算
出价水位比 β_a账户当前 oCPX 出价 / 渠道均值出价投放后台拉取
预估实消金额
Ŝ_a = B_a × η_a × γ × β_a
其中 β_a > 1 表示高于均值出价，消耗能力更强；γ > 1 表示竞争激烈，整体消耗能力下降。

校验调整规则

触发条件判断执行动作
Ŝ_a < 0.8 × B_a 消耗不足 释放超额预算（B_a − Ŝ_a × 1.1），重新注入渠道预算池，触发该渠道内账户级局部重分配
Ŝ_a > B_a 超消风险 预算上界截断至 B_a × 1.05，同时下调出价水位保护成本
0.8×B_a ≤ Ŝ_a ≤ B_a 正常 无需调整，直接写入投放平台
预算上移逻辑：释放的预算重新进入渠道级二分优化，按等边际效率原则再次分配至消耗能力更强的账户，确保渠道总预算不浪费、全局转化数最大化。

模块五：等边际效率规则约束层（修订版）

在等边际效率优化框架上叠加三条业务规则。核心修订点：规则一将动态均值基准改为历史 P50 固定基准，将点估计闸判据改为区间平均消耗率，并将"预算锁定→回收→重分配"设计为独立显式步骤，与花费预估校验模块职责完全分离。

两大机制的职责边界（核心修订）
机制处理对象触发时机决策依据输出
规则一门槛分配资格（准入控制）账户级分配之后、花费预估之前S_a(B) 花费能力曲线 + η_P50_c锁定/允许状态；释放的 ΔB
花费预估校验执行可行性（能否花出去）预算最终确定之后历史消耗率 × 流量指数 × 出价水位Ŝ_a；消耗不足/超消微调

规则一（修订版）：低跑量账户增益准入门槛

1.1 基准修订：η_P50_c 替代 η̄_c

原方案以渠道当日动态均值 η̄_c 作为判断基准，存在循环依赖——大量低效账户会拉低均值，导致阈值随之松动，低跑量账户被漏判。修订为使用渠道历史 7 日滚动 P50 消耗率作为固定参照系：

渠道历史 P50 消耗率（固定基准）
η_P50_c = P50( { η_c,d : d ∈ [D-7, D-1] } )
η_c,d = 渠道 c 在第 d 日的整体消耗率（实际消耗 / 分配预算）；P50 取7日分布中位数，不受当日低效账户拖累，稳定性高于动态均值。
受限状态触发条件（不变）
η_a < 0.8 × η_P50_c → 账户 a 标记为"受限账户"

1.2 第二关判据修订：区间平均消耗率替代一阶导数点估计

原方案用 ∂S_a/∂B（一阶导数）作为第二关判据，存在量纲混淆：点估计仅反映某一预算水位处的瞬时斜率，无法代表增量预算在整个区间内的实际消耗能力。修订为区间平均消耗率测试：

增量预算区间
ΔB_proposed = max(0, B_a_proposed − B_a_current)
B_a_proposed：账户第一轮等边际效率分配得到的理论预算；B_a_current：账户当前实际运行预算（前一日终值）
增量区间平均消耗率（第二关判据）
η_interval_a = [ S_a(B_a_current + ΔB) − S_a(B_a_current) ] / ΔB
S_a(B)：账户花费能力曲线（由历史「日分配预算→日实消」数据对拟合的 S 型函数，T+1 同步更新）
η_interval_a 表示：将 ΔB 额外预算拨给该账户，这笔钱平均能被花出去多少比例
准入条件
η_interval_a > η_P50_c
含义：增量预算的区间平均消耗率须高于渠道历史 P50 水位；低于则说明这笔预算交给该账户会大概率消耗不掉，应拒绝增配
为何区间平均优于点导数？ 一个账户可能在 B_a_current 处的瞬时边际斜率很高（点导数通过），但由于其 S_a(B) 曲线的容量上限 L 很低，ΔB 区间内的实际消耗会迅速饱和，真实区间平均消耗率远低于点估计。区间测试能捕捉这种"饱和快"的情况，更贴近实际跑量结果。

1.3 预算锁定与回收重分配（新增独立步骤）

规则一的执行结果不再依赖花费预估校验隐式处理，而是设计为账户级分配后的显式独立步骤：

账户状态判定逻辑处理方式
正常账户η_a ≥ 0.8 × η_P50_cB_a_final = B_a_proposed（保留第一轮分配结果）
受限但通过η_a < 0.8 × η_P50_c 且 η_interval_a > η_P50_cB_a_final = B_a_proposed（允许增配）
受限且锁定η_a < 0.8 × η_P50_c 且 η_interval_a ≤ η_P50_cB_a_final = B_a_current；ΔB = B_proposed − B_current 释放至回收池
预算回收与二次等边际分配
Pool = Σ ΔB_a （所有锁定账户释放的增量预算之和）
B_remaining = B_channel − Σ B_a_current（锁定账户）
对合格账户集合（正常 + 受限但通过）重新运行等边际效率二分迭代，总预算 = B_remaining
合格账户的最终预算 B_a_final 由二次分配结果决定，替代第一轮结果；整个渠道总预算 B_channel 保持不变，无预算浪费

规则二：预算与出价解耦原则（不变）

渠道总预算规模不变（ΔB_channel = 0）时，系统仅执行账户间预算再分配，严禁干预账户出价。

模型约束：S 曲线以当前出价为固定参数
Conv(B | bid = bid_fixed) 优化变量仅为预算 B，出价为外生参数
本系统负责预算层（Budget Layer），出价由 oCPX 或独立出价模块负责（Bid Layer），两层解耦，防止双向干预引发震荡。

规则三：流量分层驱动的跑量能力评估（补充与规则一的协同）

账户消耗速率受媒体流量分层与时段流量大小共同驱动，将流量指数纳入跑量能力评估。重要联动：规则一中的 η_P50_c 计算应使用流量归一化后的消耗率，避免高峰期账户因时段性消耗偏低被误触发受限状态。

流量归一化消耗率（用于规则一基准计算）
η_a_norm = η_a_raw / T_c(t_eval)
t_eval：评估时刻；T_c(t_eval)：渠道在评估时刻的流量指数；η_a_norm 消除了时段流量差异，使跨时段的跑量能力具有可比性
流量修正花费预估（Module 4 基础上修订，引入分时流量）
Ŝ_a(t) = B_a × η_a_base × T_c(t) × γ(t) × β_a
T_c(t)：当前时段流量指数；γ(t)：时段竞争度系数（随时段动态变化）；三条规则共同作用：规则三提升 η 精度 → 规则一准确识别低效账户 → 规则二保证只调预算不动出价

模块六：预算分配实验方案优化

现有 matched-pair RCT 能较好解决实验前 A/B 账户基线不一致问题，但如果实验单元长期停留在账户粒度，仍会受到媒体流量分层、账户跑量能力漂移、账户基建规模不足、渠道级预算外推不足等因素影响。优化方向是将实验从“账户对账户”升级为账户配对校准 + 账户内广告组/计划随机化 + 渠道层因果评估 + 响应曲线预算外推的混合实验框架。

核心结论：账户配对不再承担全部因果识别责任，而是作为实验前的均衡校准层；真正验证预算调节净效果时，优先在同一账户内拆分广告组/计划做随机化，最大限度控制账户质量、媒体账户权重、行业流量池与素材账户资产差异。进一步参考 Meridian 的 ROI / mROI / 响应曲线思路，将“实验验证增量效果”与“未来预算应该投向哪里”拆成两个问题：前者用 A/B、DiD、CUPED 估计净增量，后者用响应曲线和边际 ROI 判断预算是否继续加、减或转移。

6.1 现有账户粒度实验的主要偏差来源

痛点对实验结论的影响优化原则
账户跑量能力漂移即使实验前消耗、曝光、ROI 接近，实验中媒体分发、账户权重、流量层级也可能发生分化，导致 uplift 混入账户自然波动。将同账户内可比广告组/计划作为主实验单元，账户固定效应天然抵消。
账户基建成本过高若要用账户级样本验证绝对 uplift，需要大量独立账户，实际投放基建与预算成本都很高。把样本量从账户数扩展到广告组/计划数，提高实验功效。
只回答账户内预算怎么分结论偏向“一个账户内好计划多给预算”，难以回答“整个渠道下多账户怎么分预算”。实验分为计划级因果验证与渠道级外推评估两层。
外部干扰无法完全隔离大盘波动、素材上新、出价变更、定向变化会污染 ROI/CPA 对比。引入冻结窗口、协变量记录、差分中的差分（DiD）和 CUPED 降噪。

6.2 优化后的三层实验结构

校准层
继续使用账户粒度 matched-pair RCT 做实验前分层：按渠道、账户历史消耗、曝光、ROI/CPA、转化规模、账户生命周期、素材活跃度、流量层级进行匹配，保证实验账户池与对照账户池具备可比性。
主实验层
在每个入选账户内部，将广告组/计划按历史表现配对后随机分为 A/B：A 组启用预算 agent 调节，B 组保持原预算策略或仅做安全兜底。这样可在同账户内比较预算迁移带来的净增益。
外推层
在渠道维度汇总所有账户的 A/B 结果，用 DiD、合成控制或分层贝叶斯模型评估渠道级预算重分配收益，判断算法是否能支撑跨账户、跨渠道预算迁移。

6.3 推荐实验分组方式

层级实验组 A对照组 B核心控制变量
账户池入选 matched-pair 中一侧账户另一侧相似账户渠道、账户生命周期、历史消耗、ROI/CPA、转化量、素材活跃度、流量层级

推荐主评估模型：分层 DiD + CUPED 降噪
Uplift = (Y_A_after − Y_A_before) − (Y_B_after − Y_B_before)
Y_adj = Y_after − θ · (Y_before − mean(Y_before))
Y 优先选择固定 CPA/固定出价策略下的目标转化数或利润贡献；若使用 ROI、CPA 反向指标，必须确认成本口径在实验窗口内稳定。CUPED 使用实验前窗口指标降噪，适合广告指标日波动较大的场景。

6.4 预算调节动作与安全边界

  • 预算动作只作用于实验组计划：A 组计划允许在账户内预算池中互相迁移，B 组计划不参与主动迁移，避免对照组被算法污染。
  • 账户总预算保持一致：主实验阶段优先固定账户日总预算，只验证“账户内预算怎么分”；若要验证跨账户分配，进入外推层单独评估。
  • 素材与出价冻结窗口：实验前后至少设置 3～7 天冻结或记录窗口，素材大换、出价策略切换、定向变更须进入协变量表，严重变更的计划剔除或单独分层。
  • 低样本计划保护：近 7 日转化不足、预算不足或冷启动中的计划只进入观察层，不直接参与强随机，避免预算被过早剥夺。
  • 预算迁移上限：单计划日预算调整不超过昨日预算的 30%～50%，账户内 A 组总预算不突破实验前约束，防止算法引发学习期震荡。

6.5 指标体系与实验结论口径

指标类型推荐指标结论用途
主指标固定 CPA 约束下的目标转化数 uplift、利润贡献 uplift、ROI uplift、CPA 稳定性判断预算 agent 是否在不改变成本的前提下带来净增益。
过程指标预算消耗率、预算迁移金额、迁移后高效计划消耗承接率、学习期波动判断预算是否真的迁移到了可承接且有效的计划。

风险指标CPA 恶化比例、账户消耗塌陷、对照污染率、预算空转率判断是否触发回滚或缩小实验范围。

6.6 引入响应曲线的增量效果衡量

参考 Meridian 的后建模分析思路，预算实验不应只回答“实验组 ROI 是否高于对照组”，还要回答“如果未来继续增加或减少预算，增量收益会如何变化”。因此，在账户内 A/B 与渠道层 DiD 得到净增量后，需要为每个渠道、账户或计划簇拟合支出 → 增量效果响应曲线。

概念在本方案中的定义使用方式
增量效果在固定 CPA/固定出价策略下，实验组相对反事实对照多带来的目标转化、收入或利润贡献。用 A/B、DiD、CUPED 或合成控制估计，作为响应曲线的被解释变量。
ROI历史窗口内增量效果 / 实际花费，衡量过去预算是否有效。用于识别历史优质渠道或账户，但不直接作为未来加预算的唯一依据。
mROI在当前预算水平上，每新增 1 元预算带来的增量效果。用于判断是否已接近饱和；mROI 高的单元优先承接增量预算。
响应曲线在不同预算倍数下，预测增量效果如何变化的曲线。用于模拟 +10%、+20%、-10% 等预算变化后的收益和边际递减点。
响应曲线与边际收益定义（业务化表达）
IncEffect_u(B | CPA_fixed) = E[ Y_u(B, CPA_fixed) − Y_u(B_base, CPA_fixed) | controls ]
ROI_u(B) = IncEffect_u(B) / Cost_u(B)
mROI_u(B) ≈ [ IncEffect_u(B × (1 + δ)) − IncEffect_u(B) ] / [ B × δ ]
u 可以是渠道、账户、计划簇或策略桶；B_base 是对照预算或无调节预算；δ 建议取 1%～5%。ROI 看固定成本条件下的历史平均收益，mROI 看当前预算点继续加钱的边际转化收益。

6.7 响应曲线驱动的预算优化闭环

估计
先通过账户内随机化和渠道层 DiD 得到各单元的净增量效果，避免把大盘波动、素材变化、媒体分发漂移误判为预算策略收益。
拟合
将不同预算水平下的增量效果拟合为响应曲线；预算水平可来自真实实验扰动、历史自然波动、灰度扩量，或在可信区间内做小幅模拟。
决策
比较各渠道/账户/计划簇当前预算点的 mROI：mROI 高且未饱和的单元承接预算，mROI 低于目标门槛或明显低于平均 ROI 的单元减少预算。
验证
下一轮实验只放开有限预算增量，验证响应曲线预测的 uplift 与实际 uplift 是否一致，再决定是否继续扩量。
预算决策场景判定逻辑执行建议
继续加预算mROI 高于目标门槛，且响应曲线仍处于未饱和区。在安全上限内加预算，优先从低 mROI 单元迁移。
保持预算ROI 较好但 mROI 已接近门槛，继续加钱边际收益有限。保持当前预算，观察素材、出价或人群变化是否带来新承接空间。
减少预算mROI 明显低于目标门槛，且响应曲线进入边际递减区。回收预算进入渠道/账户预算池，转投更高 mROI 单元。
禁止外推历史预算几乎无波动，或模拟预算远高于历史最大预算。不直接按曲线大幅扩量，只允许小步灰度验证。
使用限制：响应曲线依赖反事实估计，且必须建立在 CPA/出价策略相对固定的前提下；若历史支出长期固定，支出为 0 或高于历史上限的点都存在外推风险；短周期 ROI 还会受到广告滞后效应影响。因此实验报告中必须同时展示估计区间、历史支出覆盖范围、CPA 稳定性、素材/出价变更记录和曲线外推倍数。
最终实验判定：若账户内计划级 A/B 显示稳定正 uplift，渠道层 DiD 结果在不同账户生命周期、素材类型、流量层级下方向一致，且响应曲线显示增量预算仍处于正 mROI 区间，则说明预算调节策略不仅能优化单账户内计划预算，也具备扩展到渠道级多账户预算分配的依据。

工程架构

系统分为离线建模与在线调度两条链路，日常以离线为主，盘中动态调整为可选增强：

离线链路（T+1 凌晨）

  • 00:25 — 两级预算优化求解
  • 00:30 — 花费预估校验 + 预算写入

  • 执行：账户级局部预算再分配
