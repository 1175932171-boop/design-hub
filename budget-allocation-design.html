信息流广告预算智能分配系统

设计方案 · v1.0  |  覆盖渠道：头条 / 广点通 / 快手  |  投放目标：APP 下载新客获取

系统概述

本方案针对信息流渠道 APP 下载拉新链路中两个核心痛点，设计一套基于算法的预算智能分配系统：

  • 渠道内账户不均衡：同一渠道下不同账户跑量效果差异显著，需基于渠道视角合理分配各账户预算；
  • 渠道间 ROI 不均衡：全局大盘不同渠道 ROI 效果不同，需基于全局视角动态分配渠道预算。

核心建设思路：将预算分配抽象为流量池最优化分解问题，通过建立"预算 - 成本 - 转化"三者的预估模型，将充沛预算优先分配给能跑量、能带来有效目标转化的账户和计划。

目标函数：在总预算约束下，最大化全链路目标转化数（新客 APP 下载激活），等价于实现最优 ROI。

整体架构

采用两级优化结构：渠道级（全局视角）先行，账户级（渠道内视角）次之，两级共享相同的 S 曲线优化框架。

数据层
账户消耗历史（1 日 / 3 日 / 7 日窗口）· 转化数据（CPA 成本、转化数）· 渠道 ROI 汇总 · 平台竞争度指数
模型层
S 曲线参数拟合（每账户 T+1 凌晨更新，参数 L / r / B₀）· 花费预估模型（消耗率 × 竞争度指数）
优化层
渠道级预算分配（全局等边际效率 · 二分迭代求解 λ）· 账户级预算分配（渠道内二分迭代，约束 [B_min, B_max]）
执行层
日预算写入调度（每日 00:30 执行）· 盘中预算动态调整（可选，每 4 小时一次）· 投放平台 API 推送

模块一：预算约束生成

每个账户的日预算取值范围 [B_min, B_max] 由三个历史消耗窗口共同决定，兼顾短期波动与长期运行稳定性：

窗口数据来源在约束中的作用
1 日（D-1）昨日实际消耗短期基准，防止单日突变
3 日近 3 日消耗均值 / 最大值中期稳定性参考，平滑偶发噪声
7 日近 7 日消耗均值 / 最大值长期运行上界，防止过度扩张

约束计算公式

预算下界（最小保障）
B_min = max( P10(近7日消耗), 3日均值 × 0.5 )
预算上界（最大允许）
B_max = min( 3日最大值 × 1.5, 7日最大值 × 1.2 )
冷启动兜底：若账户近 7 日有效消耗数据不足（新账户或冷启动期），B_min 取平台最低保护预算（建议 ≥500 元/日），B_max 取同渠道同类账户消耗分布的 P50 水位，避免新账户在竞争中被完全剥夺预算。

模块二："预算 - 成本 - 转化" S 曲线预估模型

核心假设：在固定 CPA 目标成本约束下，目标转化数随预算增大呈 S 型增长——初期边际效率高（预算充裕、流量充足），后期因竞争加剧边际效率递减，最终趋于饱和（流量容量上限 L）。

S 曲线转化预估公式

转化数预估（Logistic S 曲线）
Conv(B) = L / ( 1 + exp( −r · (B − B₀) ) )
L ：转化容量上限（历史最大转化 × 放大系数 α，建议 α = 1.2）
r ：增长斜率（拟合参数，反映预算对转化的敏感程度）
B₀ ：S 曲线拐点预算（边际效率最大处，即效率斜率 k 取最大值对应的预算点）

边际效率函数（一阶导数）

边际转化效率 ME(B)
ME(B) = dConv/dB = L·r·exp(−r·(B−B₀)) / (1 + exp(−r·(B−B₀)))²
ME(B) 在 B = B₀ 处取得最大值 L·r/4，此后随预算增大单调递减（边际递减效应）。

平均效率指标 k（ROI 斜率）

效率斜率 k(B) = 目标转化数 / 账户预算
k(B) = Conv(B) / B
最优预算点 B* 满足 k'(B*) = 0，即：ME(B*) · B* = Conv(B*)
B* 是 S 曲线上性价比最高的预算量——边际增益等于当前平均增益的交叉点。
预算低于 B*：增加预算可提升 k（物超所值区）；超过 B*：增加预算 k 下降（边际递减区）。
参数拟合方法：以账户近 30 日「日预算 → 日转化数」历史数据对，通过最小二乘法（Levenberg-Marquardt 算法）拟合 L、r、B₀ 三个参数，每日 T+1 凌晨更新一次。若有效数据点不足 7 个，退化为线性外推模型（Conv ≈ k̄·B，k̄ 为近期平均效率）。

模块三：两级最优化分配

3.1 理论基础：等边际效率原则（拉格朗日乘子法）

对于集合内任意分配单元（账户或渠道），在总预算等式约束与各单元预算区间约束下，求解最优分配：

优化目标
max Σᵢ Conv_i(Bᵢ)
s.t. Σᵢ Bᵢ = B_total
B_min,i ≤ Bᵢ ≤ B_max,i ∀ i
KKT 最优条件（等边际效率）
ME_i(Bᵢ*) = ME_j(Bⱼ*) = λ ∀ i,j（未触及约束边界的单元间）
直觉：若账户 A 的边际转化效率 > 账户 B，则从 B 抽取预算转移给 A 可提升总转化；持续调整直至两者边际效率相等，此时总转化数达到最大。

3.2 求解算法：二分法迭代（无需 LP 库）

# 输入：各分配单元的 S 曲线参数、约束区间 [B_min,B_max]、总预算 B_total
def allocate(units, B_total):

    # 给定 λ，对单元 u 反解预算（ME(B)=λ 的解，截断至约束区间）
    def solve_B(u, lam):
        B = ME_inverse(u.L, u.r, u.B0, lam)  # 解析式或数值求逆
        return clip(B, u.B_min, u.B_max)

    # 二分搜索 λ 使 ΣB_i = B_total
    lo, hi = 0, MAX_ME
    while hi - lo > EPS:
        lam = (lo + hi) / 2
        total = sum(solve_B(u, lam) for u in units)
        if total > B_total:
            lo = lam   # λ 太小 → 各单元分得预算过多 → 提高 λ
        else:
            hi = lam   # λ 太大 → 各单元分得预算不足 → 降低 λ

    return {u.id: solve_B(u, (lo+hi)/2) for u in units}

收敛条件：|ΣBᵢ − B_total| < 100 元，或迭代超 50 次取最近解。时间复杂度 O(N·log(MAX_ME/EPS))。

3.3 两级分配执行顺序

级别优化对象约束输入输出结果
第一级 渠道级 头条 / 广点通 / 快手 三渠道 各渠道历史预算上下界；总日预算 B_total 各渠道当日预算额度 B_channel
第二级 账户级 渠道内所有账户（每渠道独立求解） 各账户 [B_min, B_max]；渠道预算 B_channel 各账户当日预算 B_account
渠道聚合曲线构建：渠道级 S 曲线 = 渠道内所有账户 S 曲线的垂直叠加（在相同 λ 水平下，各账户最优预算之和 vs. 渠道总转化数），保证两级分配结果全局自洽。

模块四：花费预估与校验调整

在确定理论最优预算后，引入花费预估服务验证预算的实际可消耗性，防止"名义高预算但实际跑不出去"导致的预算浪费。

预估模型输入与输出

输入因子说明获取方式
历史消耗率 η_a近 7 日「实际消耗 / 分配预算」均值，反映账户跑量能力内部数据仓库
竞争度系数 γ渠道当日流量竞争水位（影响出价有效性）平台 API 或内部模型估算
出价水位比 β_a账户当前 oCPX 出价 / 渠道均值出价投放后台拉取
预估实消金额
Ŝ_a = B_a × η_a × γ × β_a
其中 β_a > 1 表示高于均值出价，消耗能力更强；γ > 1 表示竞争激烈，整体消耗能力下降。

校验调整规则

触发条件判断执行动作
Ŝ_a < 0.8 × B_a 消耗不足 释放超额预算（B_a − Ŝ_a × 1.1），重新注入渠道预算池，触发该渠道内账户级局部重分配
Ŝ_a > B_a 超消风险 预算上界截断至 B_a × 1.05，同时下调出价水位保护成本
0.8×B_a ≤ Ŝ_a ≤ B_a 正常 无需调整，直接写入投放平台
预算上移逻辑：释放的预算重新进入渠道级二分优化，按等边际效率原则再次分配至消耗能力更强的账户，确保渠道总预算不浪费、全局转化数最大化。

模块五：等边际效率规则约束层（修订版）

在等边际效率优化框架上叠加三条业务规则。核心修订点：规则一将动态均值基准改为历史 P50 固定基准，将点估计闸判据改为区间平均消耗率，并将"预算锁定→回收→重分配"设计为独立显式步骤，与花费预估校验模块职责完全分离。

两大机制的职责边界（核心修订）
机制处理对象触发时机决策依据输出
规则一门槛分配资格（准入控制）账户级分配之后、花费预估之前S_a(B) 花费能力曲线 + η_P50_c锁定/允许状态；释放的 ΔB
花费预估校验执行可行性（能否花出去）预算最终确定之后历史消耗率 × 流量指数 × 出价水位Ŝ_a；消耗不足/超消微调

规则一（修订版）：低跑量账户增益准入门槛

1.1 基准修订：η_P50_c 替代 η̄_c

原方案以渠道当日动态均值 η̄_c 作为判断基准，存在循环依赖——大量低效账户会拉低均值，导致阈值随之松动，低跑量账户被漏判。修订为使用渠道历史 7 日滚动 P50 消耗率作为固定参照系：

渠道历史 P50 消耗率（固定基准）
η_P50_c = P50( { η_c,d : d ∈ [D-7, D-1] } )
η_c,d = 渠道 c 在第 d 日的整体消耗率（实际消耗 / 分配预算）；P50 取7日分布中位数，不受当日低效账户拖累，稳定性高于动态均值。
受限状态触发条件（不变）
η_a < 0.8 × η_P50_c → 账户 a 标记为"受限账户"

1.2 第二关判据修订：区间平均消耗率替代一阶导数点估计

原方案用 ∂S_a/∂B（一阶导数）作为第二关判据，存在量纲混淆：点估计仅反映某一预算水位处的瞬时斜率，无法代表增量预算在整个区间内的实际消耗能力。修订为区间平均消耗率测试：

增量预算区间
ΔB_proposed = max(0, B_a_proposed − B_a_current)
B_a_proposed：账户第一轮等边际效率分配得到的理论预算；B_a_current：账户当前实际运行预算（前一日终值）
增量区间平均消耗率（第二关判据）
η_interval_a = [ S_a(B_a_current + ΔB) − S_a(B_a_current) ] / ΔB
S_a(B)：账户花费能力曲线（由历史「日分配预算→日实消」数据对拟合的 S 型函数，T+1 同步更新）
η_interval_a 表示：将 ΔB 额外预算拨给该账户，这笔钱平均能被花出去多少比例
准入条件
η_interval_a > η_P50_c
含义：增量预算的区间平均消耗率须高于渠道历史 P50 水位；低于则说明这笔预算交给该账户会大概率消耗不掉，应拒绝增配
为何区间平均优于点导数？ 一个账户可能在 B_a_current 处的瞬时边际斜率很高（点导数通过），但由于其 S_a(B) 曲线的容量上限 L 很低，ΔB 区间内的实际消耗会迅速饱和，真实区间平均消耗率远低于点估计。区间测试能捕捉这种"饱和快"的情况，更贴近实际跑量结果。

1.3 预算锁定与回收重分配（新增独立步骤）

规则一的执行结果不再依赖花费预估校验隐式处理，而是设计为账户级分配后的显式独立步骤：

账户状态判定逻辑处理方式
正常账户η_a ≥ 0.8 × η_P50_cB_a_final = B_a_proposed（保留第一轮分配结果）
受限但通过η_a < 0.8 × η_P50_c 且 η_interval_a > η_P50_cB_a_final = B_a_proposed（允许增配）
受限且锁定η_a < 0.8 × η_P50_c 且 η_interval_a ≤ η_P50_cB_a_final = B_a_current；ΔB = B_proposed − B_current 释放至回收池
预算回收与二次等边际分配
Pool = Σ ΔB_a （所有锁定账户释放的增量预算之和）
B_remaining = B_channel − Σ B_a_current（锁定账户）
对合格账户集合（正常 + 受限但通过）重新运行等边际效率二分迭代，总预算 = B_remaining
合格账户的最终预算 B_a_final 由二次分配结果决定，替代第一轮结果；整个渠道总预算 B_channel 保持不变，无预算浪费

规则二：预算与出价解耦原则（不变）

渠道总预算规模不变（ΔB_channel = 0）时，系统仅执行账户间预算再分配，严禁干预账户出价。

模型约束：S 曲线以当前出价为固定参数
Conv(B | bid = bid_fixed) 优化变量仅为预算 B，出价为外生参数
本系统负责预算层（Budget Layer），出价由 oCPX 或独立出价模块负责（Bid Layer），两层解耦，防止双向干预引发震荡。

规则三：流量分层驱动的跑量能力评估（补充与规则一的协同）

账户消耗速率受媒体流量分层与时段流量大小共同驱动，将流量指数纳入跑量能力评估。重要联动：规则一中的 η_P50_c 计算应使用流量归一化后的消耗率，避免高峰期账户因时段性消耗偏低被误触发受限状态。

流量归一化消耗率（用于规则一基准计算）
η_a_norm = η_a_raw / T_c(t_eval)
t_eval：评估时刻；T_c(t_eval)：渠道在评估时刻的流量指数；η_a_norm 消除了时段流量差异，使跨时段的跑量能力具有可比性
流量修正花费预估（Module 4 基础上修订，引入分时流量）
Ŝ_a(t) = B_a × η_a_base × T_c(t) × γ(t) × β_a
T_c(t)：当前时段流量指数；γ(t)：时段竞争度系数（随时段动态变化）；三条规则共同作用：规则三提升 η 精度 → 规则一准确识别低效账户 → 规则二保证只调预算不动出价

工程架构

系统分为离线建模与在线调度两条链路，日常以离线为主，盘中动态调整为可选增强：

离线链路（T+1 凌晨）

  • 00:00 — 数据拉取（消耗、转化、ROI）
  • 00:10 — 约束区间计算（1/3/7日窗口）
  • 00:20 — S 曲线参数更新（LM 拟合）
  • 00:25 — 两级预算优化求解
  • 00:30 — 花费预估校验 + 预算写入

在线链路（盘中动态，可选）

  • 每 4 小时检查消耗进度
  • 触发条件：实消 < 预算 × 累计消耗率 × 0.7
  • 执行：账户级局部预算再分配
  • 保护：单次调整幅度 ≤ 30%，防止剧烈波动

风险控制

核心原则：算法输出作为决策建议，所有超过阈值的预算调整操作应先在沙盒环境验证，并设置人工审核卡口，防止算法异常引发大规模亏损。
风险场景防控措施
单次预算调整幅度过大单账户日预算单次调整 ≤ 昨日预算的 50%，渠道级 ≤ 30%
S 曲线过拟合（数据稀少）有效样本 < 7 时退化为线性模型，禁止大幅扩预算
渠道 API 限流/故障写入失败时保持前一日预算，自动重试 3 次后告警
