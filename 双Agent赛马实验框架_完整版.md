# 双Agent赛马实验框架（完整版）

---

## 一、整体架构设计

### 实验背景

快手UAA是一个智能投放Agent，接入后默认开启"自动基建"和"自动调控"能力，调控动作包含：超成本关停、预算消耗控制、成本稳定控制、独立调价等。我方自研内部Agent的调控动作与UAA能力一致，为避免双重调控，需在策略执行层实现互斥，并设计科学实验方案进行UAA与内部Agent的赛马对比。

### 四组实验结构

| 实验组 | 内容说明（执行动作） | 流量分配 |
|--------|---------------------|---------|
| UAA实验组（Bucket A） | 快手UAA全托管，内部Agent熔断 | **35%** |
| Agent实验组（Bucket B） | 内部Agent托管，快手UAA不开启 | **35%** |
| 融合实验组（Bucket C） | 快手UAA开启 + Agent调控，双干预机制开启 | **15%** |
| 绝对空白组 Baseline（Bucket D） | 常规投放，不接UAA，不开Agent | **15%** |

四组之间**计划级互斥**，一个计划只归属一个Bucket，全程不漂移。

### 比较矩阵（实验价值）

| 对比 | 结论意义 |
|------|---------|
| UAA组 vs 空白组 | UAA的独立投放价值 |
| Agent组 vs 空白组 | 内部Agent的独立投放价值 |
| UAA组 vs Agent组 | **核心赛马结论**：谁的调控效果更优 |
| 融合组 vs 空白组 | 双Agent协同的综合价值 |
| 融合组 vs UAA组 | Agent在UAA基础上的增量价值 |
| 融合组 vs Agent组 | UAA在Agent基础上的增量价值 |

## 二、策略互斥机制设计

### 核心原则

> 一个计划最多只能选择一个Agent进行托管，互斥执行，避免双重调控。

### 熔断/旁路逻辑

```
内部Agent执行前置检查：
  IF 计划.uaa_status == ACTIVE（含：API查询失败时默认判定为ACTIVE）
    THEN 跳过本计划，写入旁路日志（action=bypass, reason=uaa_active）
    ELSE 执行内部Agent调控逻辑
  
  缓存策略：UAA状态本地缓存 TTL ≤ 5分钟，避免频繁API调用
  失效降级：API查询异常时，保守默认"UAA已开启"，宁可不执行，不双重调控
```

### 互斥场景说明

| 场景 | UAA状态 | 内部Agent行为 | 备注 |
|------|---------|-------------|------|
| UAA组正常态 | ON | 强制旁路 | Bucket A标准态 |
| Agent组正常态 | OFF | 正常执行 | Bucket B标准态 |
| 融合组 | ON | 双机制并行，协同调控 | Bucket C标准态 |
| 空白组 | OFF | 不介入 | Bucket D基线态 |
| 异常切换（计划误开UAA）| ON→切换 | 立即熔断，等待下一周期重判 | 保护机制 |
| UAA异常下线 | OFF | 不自动接管，人工确认后才切换 | 防止调控真空期抖动 |

### 熔断触发日志字段

```json
{
  "plan_id": "xxx",
  "bucket": "B",
  "action": "bypass",
  "reason": "uaa_active_detected",
  "ts": "2024-xx-xx xx:xx:xx",
  "agent_intended_action": "price_down_5pct",
  "uaa_cache_hit": true,
  "cache_age_seconds": 180
}
```

## 三、流量分配方案

### 入组前提（事前过滤）

在参与哈希分配之前，计划必须满足以下全部条件：

- 已度过冷启动：累计转化 ≥ 50次，且连续稳定消耗 ≥ 3天
- 实验周期内有实际消耗且已发生转化（保证可度量调控干预）

### 分层抽样维度

保证四组在以下维度的分布绝对均匀：

- 历史消耗区间（高/中/低预算段）
- 历史转化成本区间（高/中/低均包含）
- 计划创建时间（排除新老计划偏差）
- 行业/品类（排除行业基线差异）

### 哈希分配执行（稳定不漂移）

```python
bucket = hash(plan_id + experiment_id) % 100

if   bucket < 35:  assign("A")  # UAA实验组（35%）
elif bucket < 70:  assign("B")  # Agent实验组（35%）
elif bucket < 85:  assign("C")  # 融合实验组（15%）
else:              assign("D")  # 绝对空白组（15%）
```

### 实验期间动态剔除规则

| 剔除条件 | 说明 |
|---------|------|
| 被人工策略干预 | 实验期间人工调整过出价/预算的计划，全量剔除 |
| 账户预算大幅调整 | 单次调整幅度 > 50%，剔除对应污染时间段（非剔除整个计划） |
| 空耗计划 | 无消耗或无转化的计划，不参与度量评估 |

### T0分配后校验

> 分配完成后立即做分布校验：各组行业/消耗/成本分布偏差 > 10%，则重新哈希分配直至均衡。

### 对照组污染防护（双重保护）

- **系统层**：绝对空白组计划打"只读"标记，系统层面拦截批量操作
- **数据层**：记录人工干预日志，评估时剔除被干预计划

## 四、核心度量指标

### 主指标（Primary Metrics）

| 指标 | 计算方式 | 说明 |
|------|---------|------|
| 相同消耗下转化数 | Conversions / Spend（归一化） | 核心赛马指标，消除预算差异 |
| 转化成本（CAC） | Spend / Conversions | 越低越好 |
| ROI | Revenue / Spend | 有交易数据时使用 |

### 辅助指标（Guardrail Metrics，不能变差）

| 指标 | 用途 |
|------|------|
| 成本达标率 | 各组实际成本稳定在目标 ±15% 以内的计划比例 |
| 消耗达成率 | 实际消耗 / 预算目标，防止某组因调控过激导致欠跑 |
| 调控动作频次 | 对比UAA和内部Agent的介入频率是否合理 |
| 计划存活率 | 实验周期内未被关停的计划比例 |

### 诊断指标（解释原因）

- 超成本关停次数、触发频率
- 自动调价幅度分布
- 预算消耗节奏曲线（日内/周内分布）

## 五、最小样本计算与工程约束

### 样本量计算公式（双侧 t 检验）

$$n = \frac{2(Z_{\alpha/2} + Z_{\beta})^2 \sigma^2}{\delta^2}$$

### 参数取值

| 参数 | 符号 | 取值 | 说明 |
|------|------|------|------|
| 显著性水平 | α | 0.05 → 修正后 **0.0167** | 主要检验3对比较，Bonferroni修正：α/3 |
| 统计功效 | 1-β | 0.80 | Z_β = 0.84 |
| 修正后临界值 | Z_α/2 | **2.39** | 对应α=0.0167，比标准1.96更严格 |
| 最小可检测提升 | δ | **10%**（相对基线均值） | 低于此提升视为无显著业务价值差异 |
| 历史转化成本标准差 | σ | 取历史近30天各计划CAC的标准差 | 以实测数据代入 |

> **Bonferroni修正说明**：四组共有6对两两比较，主要关注3对（UAA vs 空白、Agent vs 空白、UAA vs Agent），显著性阈值调整为 α/3 ≈ 0.017，所需样本量较不修正增大约50%。结果报告同时输出95%置信区间，避免仅看p值导致"统计显著但业务无意义"的误判。

### 样本量估算示例

> 假设历史CAC均值 = 100元，σ = 30元，δ = 10%（绝对值10元）：

$$n = \frac{2 \times (2.39 + 0.84)^2 \times 30^2}{10^2} \approx \frac{2 \times 10.43 \times 900}{100} \approx 188\ 个计划/组$$

实际取值以业务历史σ代入计算，此为示意。

### 实际工程硬约束

| 条件 | 最低门槛 | 说明 |
|------|---------|------|
| 每组计划数 | ≥ **30 个** | 统计稳定性底线；理论计算值更高时以公式结果为准 |
| 每组日消耗 | ≥ **2万元** | 保证转化量可观测，低于此门槛数据噪声过大 |
| 每组累计转化数 | ≥ **100 个** | 避免小样本波动 |
| 实验持续周期 | ≥ **14天（冷启后起算）** | 覆盖至少2个完整投放节奏周期 |

> ⚠️ 融合组和空白组体量仅占15%，更易出现累计转化数不足。若T7天中检任意组累计转化 < 50个，预警并延长观察周期，**不提前下结论**。

若以上任一条件未达到，该组不进入当期评估，延长观察。

## 六、实验生命周期

```
T0  计划哈希分配 → 四组锁定，写入实验配置表，执行分布校验
 │
T1  冷启期（约3~7天）
 │  ├── UAA组：UAA自动基建 + 自动调控，内部Agent熔断
 │  ├── Agent组：内部Agent运行，UAA旁路检查激活
 │  ├── 融合组：UAA + Agent双机制并行
 │  └── 空白组：人工设置后不干预（系统只读保护）
 │
T2  冷启期结束判断（转化数达标 + 消耗稳定）
 │  └── 未达标计划继续等待，不提前纳入统计
 │
T3  正式评估期（≥ 14天，冷启后起算）
 │  ├── 每日记录：消耗、转化数、成本、调控动作日志
 │  ├── T7 中检：异常计划排查，检查各组转化数是否达标
 │  │   └── 任意组转化 < 50个 → 预警，评估是否延长周期
 │  ├── 不调整分组（中检只观察，不干预）
 │  └── 早停规则：某组成本超出目标 50% 且连续 3 天，提前终止该组，保护预算
 │
T4  结果分析
     ├── 主指标显著性检验（t检验 / Bootstrap）+ Bonferroni修正
     ├── 效应量 + 95% 置信区间报告
     ├── 辅助指标归因分析
     └── 分层分析（按行业、预算段看差异来源）
```

## 七、结果判断标准

| 结论 | 判定条件 |
|------|---------|
| UAA显著优于内部Agent | A组主指标显著优于B组（p < 0.017），且均优于空白组D |
| 内部Agent显著优于UAA | B组主指标显著优于A组（p < 0.017），且均优于空白组D |
| 两者无显著差异 | A/B组主指标差异 p > 0.017，结合辅助指标做业务判断 |
| 融合组产生协同增益 | C组显著优于A、B任意一组（p < 0.017）|
| 双Agent均不如空白组 | A、B均差于D，说明调控策略有问题，暂停迭代排查 |

### 无显著差异时的分层分析策略

> 主指标无显著差异时，按行业/预算段拆分看是否存在结构性差异，指导后续差异化策略：
> - 高预算大计划 → 适合UAA
> - 长尾小预算计划 → 适合内部Agent
> - 特定行业（如游戏/电商）→ 差异化配置

## 八、关键风险与应对

| 风险 | 应对方案 |
|------|---------|
| 对照组被污染（人工误调控）| 空白组打"只读"标记，系统层拦截批量操作；数据层记录干预日志，评估时剔除 |
| UAA悄默更新调控逻辑 | 监控UAA版本变更日志，版本切换视为实验重启信号 |
| 融合组/空白组样本不足 | T7中检预警，评估是否延长周期；不提前下结论 |
| 计划自然淘汰（跑量差被关停）| 记录计划退出时间，分析是否与调控行为相关，纳入存活率指标 |
| 分组后行业分布不均 | T0分配后立即校验，偏差 > 10% 则重新哈希 |
| UAA状态API查询失败 | 降级默认判定"UAA已开启"，保守跳过内部Agent，日志记录失败原因 |
| 大促/节假日外部冲击 | 暂停统计窗口，或将节假日期间数据单独分层，不混入主统计 |
| 早停后统计偏差 | 早停仅针对预算保护，最终分析时标注该组数据截断，采用存活分析补充 |
