🧠 百度助攻模型 · 三层建模机制
📥
样本接收层
通过观星盘 DMP 接收广告主上传的他渠道转化用户设备 ID

支持格式：IDFA / OAID / IMEI / CAID / 手机号（MD5加密）

最低门槛：100万条，建议 1~3 个月滚动更新
数据入口
🔗
行为关联层
将上传的转化用户 ID 与百度历史行为日志进行 join

确认该用户在百度有过：曝光 / 点击 / 落地 / 跳转 AppStore

确保助攻样本的"百度触达"有效性，过滤无关噪声
行为验证
🏷️
正样本标注层
助攻转化用户被标注为正样本（潜在高价值用户）

不改变归因结果，不影响媒体结算

独立于正常转化样本，作为补充训练数据

核心约束 & 注意事项
⚖️
归因不变原则
助攻样本不改变任何渠道的归因结果

转化费用仍归属原渠道（抖音/快手等）

百度仅使用数据训练模型，不参与结算计费
🔒
隐私合规要求
所有 ID 数据必须加密传输（MD5等），禁止明文

遵循个人信息保护法（PIPL）相关要求

手机号必须经 MD5 后再上传，不可还原
🎯
样本纯净度
仅回传有百度行为记录的转化用户

无百度行为的纯他渠道用户回传无效，会引入噪声

建议回传前与百度 Cookie 做预匹配验证
