二
核心引擎：目标配置驱动的三层执行模型
范式转变：搭建 Agent 摒弃「模板优先」的旧范式，确立严格执行链路：目标配置（战略锚点）→ 模板适配（战术载体）→ 账号匹配（资源绑定），三层之间形成合法域传递，下层参数不得越过上层约束。

2
投放模板（战术适配器）
在目标配置划定的框架内，定义具体的创意与投放参数
Template Adapter
🌐 全局固定字段（4项）
  • 任务类型（枚举，与目标配置锁定联动）
  • 媒体平台（头条 / 广点通 / 百度 / 快手 / B站）
  • 出价方式（下单 / 注册 / 激活 / 付费）
  • 广告类型（SDPA / 信息流，受推广目的约束）
📡 媒体相关字段（3项，动态加载）
  • RTAID：平台专属字符串（如腾讯 julang_rta_xxx / 头条 36185），按平台枚举动态刷新
  • 投放版位：按平台多选（头条：通投/站内/穿山甲；腾讯：xqxs/朋友圈…）
  • 定向包：常规/屏蔽定向包，按平台动态加载
禁止硬编码，必须动态枚举
快速填入机制：选择模板后，参数以「预览条」形式填入对话框，用户可通过自然语言微调（如「把版位改成朋友圈」）。Agent 具备增量修改能力，保留其余已配置参数不变。

三
任务流模块：结构化追溯与状态管理
任务流是 Agent 工作的结构化日志，以「任务」为维度展示，区别于对话流的非结构化信息。每个搭建任务线下包含多个投放任务，每个投放任务对应一个明确的投放目标，形成战略-战术双层追溯体系。

关键设计约束
🔗 双 ID 绑定
每个投放任务同时记录 target_config_id 和 template_id，实现战略–战术双层追溯，历史任务不受后续配置更新影响。
📊 状态可视化
🟢 成功 🟡 进行中 🔴 失败需人工 ⚪ 已取消
支持按状态筛选、快速定位异常任务并一键唤起诊断 Agent。
🔄 一对多关系
一个搭建任务线包含多个投放任务，每个投放任务对应一个明确的投放目标（任务类型 × 平台），支持平级并行执行与聚合汇总。
3+
投放模板弹窗：三环流程设计
模板弹窗是当前 Demo 最核心的交互模块，采用三环密路式设计：环1 目标配置 → 环2 账户匹配 → 环3 预算出价，每环完成后才解锁下一环，并新增 Loading 动画进度条提升体验。
🌀 环 1 — 目标配置
Step 1：点击「配置目标」按鈕，展开内联表单
任务类型 投放平台 推广目的 优化目标 出价策略 组日预算
6 个字段联动级联，非法组合实时阻断；选完后自动打标并显示符合条件的模板卡片
🌀 环 2 — 账户匹配
Step 2：模板参数填写 + 模板选择
媒体平台 出价方式 广告类型 RTAID 投放版位 定向包
投放版位采用左右联动面板； RTAID 按平台动态加载枚举；选完后 Agent 推荐匹配合规账户
🌀 环 3 — 预算出价
Step 3：确认账号 + 填写预算出价
组日预算填写 目标出价 Agent 推荐账号
单独提供「Agent 推荐」面板，展示历史账户推荐理由（资质/余额/ROI/空耗率）；支持手动剔除/追加账号；确认后自动跳转 Loading 分析页
Loading 进度动画 + 模板选择卡片
⏳ 分析中 Loading 展示
完成局环折叠后，展示带进度条的 Loading 页：

• 圆形进度圈 + 呼吸感冲波脉冲动效果
• 步骤节点：目标分析 → 模板匹配 → 账号筛选
• 自动跳转倒计时（可手动取消）
📊 模板选择卡片
分析完成后展示匹配到的模板卡片：

• 每张卡片展示：任务类型、投放平台、推广目的、优化目标、出价策略标签
• 支持单击选择，选中后边框高亮
• 选定后自动带入目标配置快照绑定

五
人机交互协同原则
🎯 人定战略，机执战术
人的职责：选择/确认目标配置，把控业务方向与风险边界。
Agent 的职责：推荐模板、匹配账号、创建计划、监控执行。
人类运营者专注于策略判断与效果验收，Agent 负责繁琐的执行细节。
✏️ 低摩擦纠错
随时可调：模板参数填入后仍可通过自然语言微调（如「把版位改成朋友圈」）。
随时可改：账号匹配后支持手动增删，Agent 实时更新计划预估数。
任何操作都可以被撤销或修改，降低犯错成本。
🔍 透明可解释
账号匹配：展示每个账号入选/排除的决策树与原因。
模板推荐：标注适配度评分与推荐依据。
数据看板：支持 L1→L3 三级下钻，每条记录可追溯。
🔔 主动预警
预算不足：账号余额预警卡片主动推送。
组合冲突：检测到非法参数组合时实时阻断并提示修复方案。
指标异常：ROI 骤降/空耗突增时主动推送告警，不等人问。
🛡️ 安全兜底
高风险确认：批量删除、大额预算变更等操作插入「待确认」节点。
内容标注：所有生成内容标注来源与版本（知识库/规则引擎/历史数据）。
权限隔离：敏感操作需账号所有者二次确认。
🔄 增量修改能力
上下文保留：Agent 切换时各自保留独立对话上下文，历史决策不丢失。
微调而非重来：自然语言微调参数时，Agent 仅修改涉及字段，保留其余配置。
变更追踪：每次调整记录 Diff 视图，便于回溯。
六
内容可追溯性体系（企业级审计）
贯穿全链路的审计能力，满足企业级合规要求。操作签名 + 版本锁定 + 决策依据 + 变更 Diff + 不可变日志五层防护，构建完整的投放操作审计体系。
🔏 操作签名
  • 每次 API 调用、计划创建均有密码学签名
  • 时间戳精确到毫秒级
  • 操作人身份绑定，不可抵赖
🔒 版本锁定
  • 目标配置与模板均绑定快照版本号
  • 历史任务不受后续更新影响
  • 可按版本号精确回溯当时的完整配置
📖 决策依据
  • 每个生成结果标注引用来源
  • 知识库片段 / 历史数据 / 规则引擎
  • 推荐理由可展开查看原始依据
🔀 变更 Diff
  • 对话微调参数时，记录变更前后对比视图
  • 支持按字段级别查看修改明细
  • 可一键回滚到任意历史版本
📦 不可变日志
  • 完整输入-输出-执行序列存入审计系统
  • 日志不可篡改，支持合规导出
  • 支持按任务线/时间/操作人多维检索
📊 审计报告导出
  • 支持按时间范围、任务类型导出完整报告
  • 包含所有决策链路与执行结果
  • 满足内外部合规审查要求

该设计将广告投放的业务复杂性封装在「目标配置」层，将媒体差异性封装在「模板适配」层，将执行风险封装在「账号匹配 + 数据看板」层

对话即投放 选择即合规 数据即信任
