1. 产品概述与目标

🎯
产品定位：搭建 Agent 是投放Agent管理平台的核心功能模块，通过 对话式 AI 工作流，将广告搭建任务（目标配置 → 模板匹配 → 账户诊断 → 预算出价 → 广告创建）全链路自动化，大幅降低投放人员的操作门槛和重复工作量。

核心价值

维度传统方式搭建 Agent 方式提升效果
操作路径多平台逐账户手动配置一次对话完成全链路步骤减少 80%+
账户召回人工查找匹配账户基于映射表智能召回准确率提升
模板复用每次重新填写模板参数预置目标配置一键选择效率提升 3x+
可追溯性无完整操作记录对话工作流完整回放100% 可审计

用户角色

角色核心诉求主要使用场景
投放运营（主用户）快速批量搭建广告，减少重复劳动日常广告计划创建
投放管理者查看搭建记录、审计操作行为搭建记录查看与分析

2. 搭建 Agent 对话工作流程

💬
搭建 Agent 采用 对话式（Conversational UI）工作流，用户与 Agent 交替进行 8 轮核心对话，分3个阶段完成广告搭建。对话框固定在页面底部（chat-active 模式），对话消息区域在上方滚动展示。

Step 3 / 用户

各步骤详细说明

步骤发起方内容描述触发动作备注
Step 1 用户 点击「使用模板创建广告」按钮，发起搭建请求 openTplConv()，进入 heroConvMode 对话框固定底部，历史消息区在上方滚动
Step 2 Agent 展示预置目标配置宫格（最多6个 TARGET_CONFIGS 卡片），附打字机动画输出说明 heroStartRound1()，convRound2() 卡片含任务类型/渠道/平台/优化目标/出价方式/日预算
Step 3 用户 点击目标配置卡片，写入 tplState，Agent 确认并输出已选配置摘要 selectConvTargetConfig(id) 可在此阶段切换配置；选中后卡片高亮
Step 4 Agent 根据目标配置的 rtaid 匹配 AD_TEMPLATES，输出项目模板 + 投放模板列表（宫格展示） convRound4()，凭 targetConfigId 查询模板 分阶段分割线：「Agent 开始匹配广告模板」
Step 5 用户 选择投放模板（placTpl），Agent 记录模板参数写入 tplState selectConvTpl() 模板选择联动版位/定向/广告类型
Step 6 Agent 基于 MAPPING_TABLE 筛选匹配账户，输出账户诊断表格（前60%推荐/后40%谨慎），展示日均消耗/ROI convRound6()，convRound7(allRows) 分阶段分割线：「Agent 进行账户诊断」；支持点击「+ 添加账户」补充
Step 7 用户 勾选确权账户，填写组日预算、目标出价，点击「确认账户」 heroSubmitAcctForm() 校验：账户≥1；日预算≥1000元；目标出价非空且≥1元
Step 8 Agent 2.5s 延迟后：✅ 完成自运营模板创建提示；自动弹出 tplConfirmPanel 展示模板详情确认 openTplConfirmPanel()；等待用户确认后 convStartAdCreation() 点击「是，立即创建」：关闭 confirmPanel，停留对话页，展示已创建广告账户链接
⚠️
重要设计原则：点击「是，立即创建广告」后，页面 保持在对话页面（heroConvMode = true），不跳转回首屏。Agent 在当前对话区输出创建结果，展示已创建广告的账户 ID 链接，点击账户 ID 可在右侧打开「已创建广告」面板。

4. M2 目标配置模块（TARGET_CONFIGS）

🎯
目标配置是搭建流程的第一步，系统内置6套预置配置（TARGET_CONFIGS），用户选中后可自动跳过手动逐字段配置，一键关联 RTAID、广告类型、版位、定向等核心参数。

taskTypestring任务类型，定义推广业务目标是1688主客拉新

optGoalstring优化目标（转化目标）是app内付费 / 激活 / 注册 / 下单
bidStrategystring出价策略是稳定成本 / 最优成本 / 最大转化

5. M3 模板选择模块

📋
在目标配置确认后，Agent 基于 AD_TEMPLATES 数据（按 rtaid 索引）输出匹配的自运营模板，分为项目模板（projTpl）和投放子模板（placTpls）两层结构。

模板数据结构字段

字段类型含义示例
projTplstring项目模板名称（顶层模板）sdpa-头条-拉新-36185-安卓付费-站内-测爆
placTplsstring[]投放子模板列表（版位层级模板）['站内-新素材', '站内-高质', '站内-昨消100']
adTypestring广告展现形式sdpa / 信息流
placementsstring[]关联的投放版位['站内']、['朋友圈','视频号']
targetingstring关联定向包名1688主客拉新_拉新-安卓-站内-内容素材
rtaidstring关联 RTA 策略 ID36185
displaystring在宫格中展示的简短描述头条·站内·sdpa·安卓付费

模板宫格交互规范

  • 展示Agent 输出多个模板卡片（宫格布局），每卡片包含项目模板名 + 投放子模板列表（圆点式展示）
  • 交互点击卡片选中，写入 tplState.projTpl / placTpls / adType / placements / targeting
  • 自动选中后，用户发送确认消息，Agent 进入账户诊断阶段（分割线：「Agent 进行账户诊断」）

6. M4 账户诊断与确权模块

🏦
Agent 基于 MAPPING_TABLE（账户映射表）多维筛选出匹配账户，并输出诊断结果表格。前60%账户标记为「推荐✅」，后40%标记为「谨慎⚠️」，用户勾选确权账户。

MAPPING_TABLE 字段说明（账户映射表）

字段名类型含义用途示例
accIdstring广告账户 ID账户唯一标识，用于召回和确权61645013
rtaidstringRTA 策略 ID第一维筛选条件，与目标配置关联julang_rta_20241211200006565
platformstring媒体渠道平台第二维筛选，过滤媒体属性广点通 / 头条 / 百度 / 快手 / B站
taskstring任务类型第三维筛选，对应目标配置 taskType1688主客拉新 / 拉新AAC
bidMethodstring出价方式/优化目标精确匹配转化目标首次付费 / 激活 / 下单 / 注册
placementstring投放版位版位维度筛选腾讯平台与内容媒体 / 朋友圈 / 站内
adTypestring广告类型广告类型精确匹配sdpa / 信息流
osstring操作系统区分安卓/iOS账户安卓 / iOS
targetIdstring定向 ID定向包绑定标识1026 / 1271 / 654
targetingstring定向包名称定向包精确匹配1688主客拉新_拉新-安卓-常规定向-优量汇屏蔽

账户筛选逻辑（多维联合过滤）

// filterMappingTable() — 多维度联合过滤 MAPPING_TABLE.filter(row => { if (taskType && row.task !== tplState.taskType) return false; if (platform && row.platform !== tplState.platform) return false; if (rtaid && row.rtaid !== tplState.rtaid) return false; if (adType && row.adType !== tplState.adType) return false; if (os && row.os !== tplState.os) return false; if (placements.length && !placements.includes(row.placement)) return false; if (bidMethod && row.bidMethod !== tplState.bidMethod) return false; if (targeting && row.targeting !== tplState.targeting) return false; return true; })

列名含义展示规则

账户 ID广告账户数字 ID直接展示，支持复制

批量添加账户（addAccPanel）

➕
当 Agent 推荐的账户不满足需求时，用户可通过右侧「添加账户面板」补充账户。
字段控件类型约束规则
账户 ID 输入框textarea（多行文本）仅接受纯数字；多账户用逗号（,、，）或换行分隔；正则校验 /^\d+$/
操作系统单选 Chip安卓（默认选中）/ iOS
  • 示例输入「92398048，38492008，38249047」→ 解析为 3 个账户 ID 批量加入候选池

  • 限制非数字字符（字母、符号）自动过滤忽略

7. M5 预算与出价模块

字段字段名类型约束规则提示文案
组日预算 tplDailyBudget number（整数，元/天） 必填；最低 1000 元/天；建议值来自目标配置 dailyBudget 低于 1000 元时显示「建议日预算 ≥ 1000 元」警告
目标出价 tplTargetBid number（浮点数，元/次） 必填；建议 ≥ 1 元；允许小数精度 0.1 ≥ 1元 显示绿色「≈ X.XX 元/次」；0~1 显示橙色「建议出价 ≥ 1 元」

校验逻辑

  • 必填确认账户数量 ≥ 1 个（需先勾选账户）
  • 必填组日预算 ≥ 1000 元，不满足时阻断提交并 alert 提示
  • 必填目标出价非空，且 > 0（建议 ≥ 1 元）
  • 提示三项均填写后，显示汇总说明：「已选 N 个账户，组日预算 X 元，目标出价 Y 元 —— 可确认填入对话框 ✅」

面板联动左移规则（_shiftFixedElems）

  • 自动任意面板打开时调用 _shiftFixedElems(true)：superCard 和 chatArea 向左偏移
  • 自动所有面板关闭时调用 _shiftFixedElems(false)：恢复原始位置
  • 参数startL = 100px（导航栏60px + 水平内边距40px），chatW ≤ 860px 时计算偏移量
  • 注意若还有其他面板开着，不还原左移（多面板判断逻辑）
